THREE LAYERS06 / 12
METHOD · VISUAL · RUNTIME

它不是只会排版，
而是同时处理三层问题。

三层彼此独立：故事方法不等于视觉皮肤，视觉系统也不应该破坏浏览器运行与离线交付。

01 · METHOD故事

从日常观察出发，写出 reframe、证据与观众最终应该带走的判断。

02 · VISUAL视觉

用纸色、网格、字体角色和克制的强调色建立统一阅读气质。

03 · RUNTIME交付

单 HTML、横向切页、语义动效、索引、静态模式和失败降级。

方法决定讲什么视觉与运行时决定如何被理解
WORKFLOW07 / 12
FIVE DECISIONS BEFORE DELIVERY

Agent 不是直接写页面，
而是连续做五次判断。

每一步都有明确输出，避免在视觉完成后才发现故事顺序错了。

01理解材料

观众是谁，分享后希望发生什么。

02写感知简报

观察、感受、重构与余味。

03安排故事

进入、细节、证据、方案与决定。

04选择页面

每页只承担一个认知任务。

05渲染检查

切页、动效、字体、截断和降级。

先建立故事，再生成视觉最后检查真实渲染结果
VISUAL SYSTEM08 / 12
ONE PAPER · ONE ACCENT · THREE ROLES

风格不是“米白＋宋体”，而是每个视觉选择都有职责。

TYPE ROLES

衬线负责观察，黑体负责工作。

SERIF进入、用户原话、reframe、余味
SANS分析、证据、产品、决策
MONO时间、页码、样本与方法标签
COLOR ROLES

一种纸色，一个强调语义。

白不是空背景，而是纸、光和接受信息的空间；红褐只标记被发现的痕迹、选择或最终落点。

约束：不为了“高级感”增加第四套字体、多种亮色或无意义装饰。

视觉角色来自内容角色克制不是减少，而是让关系更清楚
SEMANTIC MOTION09 / 12
MOTION FOLLOWS READING

NOT A GENERIC FADE-UP

动效不是让页面更炫，
而是决定先看什么。

切页负责空间移动；标题、证据、界面和注释按页面任务依次成立。回到旧页会重播，快速翻页会取消旧动画。

FIT11 / 12
KNOW THE BOUNDARY

它适合有观点的产品故事，不适合把所有信息塞进一页。

GOOD FIT

设计评审、产品发布、策略叙事与创意提案

适合产品经理、设计师、独立开发者和需要有表达气质的团队汇报。

NOT THE BEST FIT

密集培训、财务表格与多人实时编辑

如果必须交付可编辑 PPTX、塞入大量表格或多人同时改稿，应选择其他工具。

它的选择：用更少页面对象，换取更清楚的观点、节奏和阅读顺序。

明确不适合什么，才能知道它真正擅长什么产品边界也是设计的一部分
